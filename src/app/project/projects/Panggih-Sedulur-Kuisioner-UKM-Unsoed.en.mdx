# Panggih Sedulur Apps

**Panggih Sedulur Apps** is an interactive aptitude questionnaire platform for Unsoed Student Clubs (UKM). This system integrates a biodata form, an aptitude test with _branching logic_, and accurate personalized recommendation calculations using Supabase integration.

---

## 🚀 Tech Stack

This system is developed with a modern _tech stack_ to ensure fast performance and an optimal user experience:

- **Framework**: Next.js (App Router)
- **Styling**: Tailwind CSS
- **Form & Validation**: React Hook Form + Zod
- **Database**: Supabase
- **Language**: TypeScript

---

## 🎯 Main Categories & Logic Flow

The questionnaire's _scoring_ system is focused on 5 main categories at _scoring level 1_:

1. **Sports**
2. **Arts**
3. **Reasoning**
4. **Nature Enthusiasts**
5. **Social & Discipline**

_(Note: Categories such as Religion, KIP-K scholarship paths, and Faculty specifications are applied dynamically as pre-filters based on the user's biodata input)._

### Questionnaire Implementation Flow:

1. **Biodata Submission**: Users fill out a form at `/kuisioner/biodata`, which is persistently stored in the browser session (localStorage).
2. **Level 1 Questions**: Users answer 5 initial questions that determine the base score for each category.
3. **Branching Logic**: The system detects the top 2 categories from the _level 1 scoring_ results. Then, it intelligently displays follow-up questions (levels 2 and 3) sequentially, specifically for those top categories.
4. **Recommendation Calculation**: The best UKM recommendations are generated by combining:
   - Answers from the _branching_ logic (most specific to their interests)
   - _Pre-filter_ rules from the biodata (Religion, KIP-K, Faculty)
   - _Fallback scoring_ if the _branching_ flow does not yield a specific recommendation.
5. **Data Storage**: All test results and detailed UKM recommendations are recorded in the `student_results` table in the **Supabase** database.
6. **Result Page**: Users are immediately redirected to `/kuisioner/result` to interactively view their recommendation results.

## 📁 Implementation Structure

The system is organized into efficient modules:

- **Data & Validation**:
  - `lib/schema.ts` (Data structure validation using Zod)
  - `lib/questions.ts` (Question bank and _branching logic_ scenarios)
  - `data/UkmLogic.ts` (Specific recommendation algorithms per UKM)
- **Interactive Components**:
  - `BiodataForm.tsx` (Handling interactive user forms)
  - `QuestionCard.tsx` & `ProgressBar.tsx` (Interactive test UI)
  - `UkmCard.tsx` (Visualizing detailed recommendation results)
- **Backend & Database**:
  - `lib/supabase.ts` (Supabase _client_ configuration and integration)
  - Reliable asynchronous synchronization.
